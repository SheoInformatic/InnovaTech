Common questions from professors:
✓ "Why Kubernetes instead of ECS?"
  → Portability, community, standardization, multi-cloud ready
✓ "How does autoscaling work?"
  → HPA monitors metrics, scales pods based on thresholds
✓ "How do services communicate?"
  → DNS discovery within cluster, Orders calls Products via HTTP
✓ "What happens if a pod dies?"
  → Kubernetes auto-restarts it, load continues
✓ "How does CI/CD work?"
  → GitHub Actions builds images, pushes to ECR, kubectl applies manifests
✓ "What about data persistence?"
  → MySQL uses PersistentVolume, survives pod restarts
✓ "How is it secure?"
  → Network policies, RBAC, secrets encryption, no hardcoded credentials
